How to see when a user ran a report for the last time in Salesforce

Nov 11, 2018

We can query RecentlyViewed SOQL object in the context of a specific user:

RecentlyViewed does not have any column which tells about what user it’s about - it always returns data about the user that performs the query. Id contains the id of the recently viewed record (Report in our case), Name contains the record’s name and LastViewedDate contains the date when the record was recently viewed by current user. It turns out that when a user runs a report manually, SFDC marks the Report’s SOQL object as recently viewed. Thanks to that, LastViewedDate from RecentlyViewed object of Type ‘Report’ will contain the date time of when the current user ran the report.

Since RecentlyViewed about a specific user is queryable only for that user, we need to find a way to run (in the context of that user) some code that will perform a SOQL query on RecentlyViewed and copy the information into a custom object. One way to accomplish that would be to create a Lightning Component which would on init call an Apex method that will perform these tasks.

Now we can place the component on the landing page that the users see when they log into Salesforce, or some other page that is frequently visited by users. The component has no body, so it will be invisible, but it will execute the Apex code when a user visits a page with it.

After that, we’ll be able to start gathering data about which user ran which report. The administrator wil be able to create another report on the custom object RecentlyRunReport__c grouped by user or report name.
